pyptx is a Python DSL for writing handcrafted PTX kernels targeting NVIDIA Hopper (sm\_90a) and Blackwell (sm\_100a) GPUs. Every `ptx.*` call emits exactly one PTX instruction — no compiler, no scheduler, no optimizer between your Python and the assembly it produces. The same kernel object runs from `jax.jit`, PyTorch eager, and `torch.compile`.

## What pyptx does

pyptx turns a Python function into a PTX assembly string. During tracing, `reg.*` allocates registers, `smem.*` describes shared-memory regions and barriers, and `ptx.*` records instructions. The result is PTX you can inspect before it ever touches the driver.

```python
from pyptx import kernel, reg, smem, ptx, Tile
from pyptx.types import bf16, f32

@kernel(
    in_specs=(Tile("M", "K", bf16), Tile("K", "N", bf16)),
    out_specs=(Tile("M", "N", f32),),
    grid=lambda M, N, K: (N // 64, M // 64),
    block=(128, 1, 1),
    arch="sm_90a",
)
def gemm(A, B, C):
    sA = smem.wgmma_tile(bf16, (64, 16), major="K")
    sB = smem.wgmma_tile(bf16, (16, 64), major="MN")
    acc = reg.array(f32, 32)
    # each ptx.* call emits exactly one PTX instruction
    # ptx.wgmma.mma_async(shape=(64, 64, 16), ...)

print(gemm.ptx(M=4096, N=4096, K=4096))
```

## One kernel, three runtime paths

```python
# PyTorch eager
out = gemm(a, b)

# torch.compile
out = torch.compile(gemm)(a, b)

# JAX jit
out = jax.jit(gemm)(a, b)
```

## Getting started

<Steps>
  <Step title="Install pyptx">
    ```bash
    pip install pyptx
    ```
    Add runtime support for your framework:
    ```bash
    pip install 'pyptx[torch]'   # PyTorch eager + torch.compile
    pip install 'pyptx[jax]'     # jax.jit via typed FFI
    pip install 'pyptx[all]'     # both
    ```
  </Step>
  <Step title="Write your first kernel">
    Define a `@kernel` function using `reg`, `smem`, and `ptx` namespaces. Each `ptx.*` call emits exactly one PTX instruction.
  </Step>
  <Step title="Inspect the PTX">
    Call `kernel.ptx()` to see the exact assembly your Python function produced before running it on hardware.
  </Step>
  <Step title="Launch from JAX or PyTorch">
    Pass your kernel to `jax.jit`, `torch.compile`, or call it directly as a PyTorch eager op.
  </Step>
</Steps>
